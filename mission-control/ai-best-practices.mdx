# AI Coding Best Practices for LayerFive

This document outlines best practices for using AI coding assistants effectively and responsibly in the LayerFive project.

## Core Principles

### 1. AI as Assistant, Not Replacement
- **You are the developer**: AI provides suggestions, you make decisions
- **Critical thinking required**: Always review and understand AI-generated code
- **Domain knowledge essential**: AI doesn't know your business logic
- **Quality ownership**: You're responsible for code quality, not the AI

### 2. Context is Everything
The quality of AI output depends on the context you provide:
- Project structure and conventions
- Related code and dependencies
- Business requirements
- Technical constraints
- Existing patterns to follow

### 3. Iterative Collaboration
- Start with high-level design
- Break down into smaller tasks
- Review each piece before proceeding
- Refine and improve incrementally

## Code Generation Best Practices

### Provide Clear Requirements
```
❌ Bad:
"Create a user model"

✅ Good:
"Create a Django User model in layerfivecore/useraccount/models.py that:
- Extends AbstractUser
- Adds company (FK to Company model)
- Adds role (choice field: admin, user, viewer)
- Has created_at and updated_at timestamps
- Includes proper string representation"
```

### Request Specific Implementations
```
❌ Bad:
"Make it work with the database"

✅ Good:
"Use Django ORM with select_related('company') to avoid N+1 queries
when fetching users with their company information"
```

### Include Error Handling Requirements
```
✅ Good prompt:
"Create this endpoint with error handling for:
- Invalid input (400)
- Unauthorized access (401)
- Not found (404)
- Database errors (500)
Include appropriate log messages"
```

## Security Best Practices

### Always Review for Security Issues
Check AI-generated code for:
- [ ] SQL injection vulnerabilities
- [ ] XSS vulnerabilities
- [ ] Authentication/authorization gaps
- [ ] Sensitive data exposure
- [ ] Insecure dependencies
- [ ] Hard-coded secrets
- [ ] Weak cryptography

### Example Security Review
```python
# ❌ AI might generate (insecure):
query = f"SELECT * FROM users WHERE id = {user_id}"

# ✅ You should change to:
user = User.objects.get(id=user_id)  # Parameterized, safe
```

### Secure Coding Checklist
- [ ] No hard-coded credentials or API keys
- [ ] Input validation on all user data
- [ ] Proper authentication on endpoints
- [ ] Authorization checks before data access
- [ ] SQL injection prevention (use ORM)
- [ ] XSS prevention (escape output)
- [ ] CSRF protection enabled
- [ ] Secure password handling

```python
    """
    Process and aggregate customer data for reporting.

    Args:
        customer_id: The unique identifier of the customer
        include_history: Whether to include historical data (default: False)

    Returns:
        Dictionary containing:
        - customer: Customer details
        - metrics: Aggregated metrics
        - history: Historical data (if include_history=True)

        ValueError: If customer_id is invalid

        >>> print(data['metrics']['total_purchases'])

    """
```

## Django-Specific Best Practices

### Model Best Practices
- Use appropriate field types
- Add db_index for frequently queried fields
- Implement clean() for validation
- Use validators from django.core.validators
- Add Meta class with ordering, constraints
- Implement __str__ method

### API Best Practices
- Use DRF ViewSets for CRUD
- Implement proper permissions
- Add pagination to list views
- Use appropriate status codes
- Validate input with serializers
- Document with docstrings

### Database Best Practices
- Always create migrations
- Review migration files
- Use transactions for related operations
- Optimize queries with select_related/prefetch_related
- Add indexes for foreign keys

## Angular-Specific Best Practices

### Component Best Practices
- Use OnPush change detection
- Implement OnDestroy and unsubscribe
- Keep templates simple
- Use smart/dumb component pattern
- Avoid logic in templates

### Service Best Practices
- Use HttpClient for API calls
- Implement error handling
- Return Observables
- Use RxJS operators for transformation
- Cache when appropriate

### Performance Best Practices
- Lazy load modules
- Use trackBy in ngFor
- Unsubscribe from observables
- Minimize change detection
- Optimize bundle size

## Common AI Pitfalls to Avoid

### 1. Over-Reliance
Don't blindly trust AI output:
- Always review generated code
- Understand what the code does
- Verify it meets requirements
- Test thoroughly

### 2. Insufficient Context
Provide enough context:
- Project structure
- Existing patterns
- Related code
- Business rules
- Technical constraints

### 3. Ignoring Edge Cases
AI often generates happy-path code:
- Request edge case handling
- Add null checks
- Validate inputs
- Handle errors

### 4. Copy-Paste Without Understanding
Never commit code you don't understand:
- Read through generated code
- Ask AI to explain unclear parts
- Refactor if needed
- Add comments for complex logic

### 5. Security Oversights
AI might generate insecure code:
- Review for vulnerabilities
- Validate all inputs
- Check authentication/authorization
- Don't hard-code secrets
