## Overview

LayerFive uses AI coding assistants (Claude Code, GitHub Copilot, etc.) to accelerate development while maintaining superior code quality. This page outlines **mandatory compliance requirements** and links to detailed resources.

<Warning>
**Compliance is Non-Negotiable**: All developers must follow the requirements outlined in this guide and the linked resources before using AI coding assistants.
</Warning>

## Mandatory Requirements

### 1. Co-Author Attribution

Every commit with AI-generated code **MUST** include co-author attribution for Swarmia tracking:

```bash
git commit -m "feat: Add customer export API

Implemented CSV and Excel export formats.

Co-Authored-By: Claude <noreply@anthropic.com>"
```

<Info>
This attribution enables tracking AI impact in Swarmia and helps measure development velocity improvements.
</Info>

### 2. Pull Request Disclosure

Every PR with AI-generated code **MUST** use the [PR template](/ai/PR-TEMPLATE.md) and include:

- List of AI-generated components with % contribution
- List of manually written components
- Areas flagged for extra review
- Changes made after AI generation
- Development metrics (time saved, debugging time, net saved)

**Example PR Disclosure:**

```markdown
## AI Usage Disclosure

### AI-Generated Components
- Models: ReportTemplate (95% AI)
- Serializers: ReportTemplateSerializer (100% AI)
- Views: ReportTemplateViewSet (80% AI, manually added permissions)
- Tests: test_report_template.py (90% AI)

### Manual Components
- URL routing
- Migration review
- Integration with existing reports page

### Review Focus Areas
⚠️ Permission logic (lines 45-62) - please verify authorization
⚠️ Template sharing (lines 120-145) - complex logic needs review

### Metrics
- Estimated without AI: 8 hours
- Actual with AI: 3 hours
- Net time saved: 3.5 hours
- AI contribution: 75%
```

### 3. Protected Areas Compliance

**NEVER** use AI in protected areas without senior approval. See [PROTECTED-AREAS.md](/ai/PROTECTED-AREAS.md) for the complete list.

<Warning>
**Critical Areas (Manual Development Strongly Recommended)**:
- 🔴 Authentication/Authorization logic
- 🔴 Payment/Financial code
- 🔴 PII/Privacy-sensitive code
- 🔴 Production infrastructure
- 🔴 Security/Encryption code
- 🔴 Database migrations affecting production data
</Warning>

### 4. Metrics Tracking

Track and report for every AI-assisted PR:

- **Time estimated without AI**: How long would this take manually?
- **Actual time with AI**: Wall-clock time from start to PR
- **Time saved**: Estimated - Actual
- **Review time**: Time reviewing AI-generated code
- **Debugging time**: Time fixing AI code issues
- **Net time saved**: Time saved - Review time - Debugging time

<Info>
See [METRICS.md](/ai/METRICS.md) for detailed tracking guidance and examples.
</Info>

### 5. Quality Standards

You are responsible for ALL code you commit, regardless of who/what wrote it:

- [ ] Reviewed every line of AI-generated code
- [ ] Understand what the code does
- [ ] Security review completed
- [ ] Tests written and passing
- [ ] No hard-coded secrets
- [ ] Follows LayerFive conventions

## Review Requirements

Different types of code require different review processes:

### Standard AI-Generated Code
- 1 approving reviewer
- Developer reviewed all code
- All tests passing
- Security checklist completed

### High-Risk Areas
- 2 approvers (at least 1 senior)
- Extra scrutiny on complex logic
- Performance testing if applicable

### Protected Areas
- 2 senior approvers required
- Security specialist review
- Comprehensive testing
- Documented justification for AI usage

## Outage Attribution

<Warning>
**CRITICAL**: Any production outage must be analyzed for AI involvement.
</Warning>

If an outage occurs:

1. Determine if AI-generated code was involved
2. Analyze what was AI-generated vs manual
3. Review the approval and review process
4. Identify what was missed in review
5. Document learnings and prevention steps
6. Update protected areas if needed

**Zero tolerance**: Repeated AI-related production issues will trigger policy review.

## Getting Started

### For New Developers

**Before using AI** (non-negotiable):

1. Read [COMPLIANCE.md](/ai/COMPLIANCE.md) completely
2. Review [PROTECTED-AREAS.md](/ai/PROTECTED-AREAS.md)
3. Review [METRICS.md](/ai/METRICS.md)
4. Acknowledge compliance requirements to tech lead

**Learning the workflow**:

5. Review agent file for your role ([backend](/ai/backend-agent.md) or [frontend](/ai/frontend-agent.md))
6. Browse [prompt templates](/ai/prompt-templates.md)
7. Complete first task with mentor supervision
8. Create first PR using the [PR template](/ai/PR-TEMPLATE.md)

### For Experienced Developers

1. Review [COMPLIANCE.md](/ai/COMPLIANCE.md) if you haven't
2. Use [PR-TEMPLATE.md](/ai/PR-TEMPLATE.md) for all AI PRs
3. Track metrics honestly and completely
4. Share effective patterns with team

## Success Metrics

We measure AI development success across three dimensions:

### Velocity
- **Target**: 20-40% time savings on feature development
- **Goal**: Faster than competitor tech agencies
- **Tracking**: Sprint velocity, time-to-market

### Quality
- **Target**: AI code bug rate ≤ manual code bug rate
- **Critical**: Zero production outages from AI code
- **Coverage**: ≥80% test coverage on AI-generated code

### Compliance
- **Target**: 100% AI attribution in commits
- **Target**: 100% PR disclosure completion
- **Target**: Zero violations of protected areas

## Monthly Reporting

### Individual (All Developers)

At the end of each month, compile your AI usage:

- Total PRs with AI contribution
- Total time saved vs debugging time
- Net time saved
- Quality metrics (bugs, coverage)
- Learnings and patterns discovered

### Team (Tech Lead)

Quarterly comprehensive analysis:

- Aggregate team metrics
- ROI analysis (time saved vs tool cost)
- Quality comparison (AI vs manual)
- Swarmia analytics review
- Competitive positioning
- Process improvements

## Enforcement

### Violations

| Severity | Examples | Response |
|----------|----------|----------|
| **Minor** | Missing co-author attribution, incomplete metrics | PR rejected, must update |
| **Major** | AI in protected areas without approval, hiding AI usage | Tech lead discussion, potential AI privilege loss |
| **Critical** | AI-caused outage due to negligence, repeated violations | Formal performance review |
