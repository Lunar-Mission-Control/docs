# Any-Click vs Last-Click Attribution: The Double Counting Problem

## TL;DR

**Does Any-Click double count?** ✅ **YES - BY DESIGN!**

**Should we use Last-Click instead?** ⚠️ **DEPENDS** - Last-Click is more conservative, but both have use cases.

---

## The Real Example: Order #5678 ($100)

### Customer Journey:
1. **Feb 1** - Clicks Google Ad → browses site
2. **Feb 8** - Opens email → browses site
3. **Feb 20** - Clicks Facebook Ad → **PURCHASES $100**

## Step 1: Order Credit Table (`eventapp_ordercredit`)

**File:** `eventapp/management/commands/process_ordcredit.py:93-115`

When the order is processed, **6 rows** are created (one per touchpoint):

```sql
id  | tr_orderid | source_credit  | tr_total | credit_score | credit_event_date
----|------------|----------------|----------|--------------|-------------------
101 | 5678       | Google Ads     | 10000    | 0.200        | 2026-02-01 10:00
102 | 5678       | Google Ads     | 10000    | 0.200        | 2026-02-01 10:05
103 | 5678       | Email          | 10000    | 0.200        | 2026-02-08 14:30
104 | 5678       | Direct         | 10000    | 0.000        | 2026-02-15 09:00  ← Excluded
105 | 5678       | Facebook Ads   | 10000    | 0.200        | 2026-02-20 16:00
106 | 5678       | Facebook Ads   | 10000    | 0.200        | 2026-02-20 16:10
```

**Key:** The order total (`tr_total = 10000` = $100) is **repeated on every row**.

## Step 2: How ANY-CLICK Aggregates This

**File:** `interaction_insight/management/commands/ProcessInsightSummary.py:186-202`

```sql
SELECT
    sum(tr_total) as any_click_revenue,
    source_credit
FROM eventapp_ordercredit
INNER JOIN (
    -- Get the LATEST touchpoint for EACH source+order combination
    SELECT max(id) as maxid, tr_orderid, source_credit
    FROM eventapp_ordercredit
    WHERE event_date = '2026-02-20'
    GROUP BY tr_orderid, source_credit  -- ← Key: Groups by BOTH order AND source
) ON maxid = id
WHERE source_credit != 'Direct' OR credit_score != 0
GROUP BY source_credit
```

**What this does:**

1. **Groups by `tr_orderid, source_credit`** → One row per source per order
2. Takes `max(id)` → Latest touchpoint for each source
3. Selected IDs: **102** (Google), **103** (Email), **106** (Facebook)
4. Sums `tr_total` for each source

**Result:**

```sql
source_credit  | any_click_revenue
---------------|-------------------
Google Ads     | 10000   ($100)  ← Full order value
Email          | 10000   ($100)  ← Full order value
Facebook Ads   | 10000   ($100)  ← Full order value
---------------|-------------------
TOTAL          | 30000   ($300)  ← 3x the actual order!
```

**This is shown on Revenue Comparison page!**

## Step 3: How LAST-CLICK Aggregates This

**File:** `interaction_insight/management/commands/ProcessInsightSummary.py:219-232`

```sql
SELECT
    sum(tr_total) as last_click_revenue,
    source_credit
FROM eventapp_ordercredit
INNER JOIN (
    -- Get the LATEST touchpoint for the ENTIRE order
    SELECT max(id) as maxid, tr_orderid
    FROM eventapp_ordercredit
    WHERE event_date = '2026-02-20'
      AND (source_credit != 'Direct' OR credit_score != 0)
    GROUP BY tr_orderid  -- ← Key: Groups ONLY by order (not source)
) ON maxid = id
GROUP BY source_credit
```

**What this does:**

1. **Groups by `tr_orderid` ONLY** → One row per order (not per source)
2. Takes `max(id)` → Latest touchpoint across ALL sources
3. Selected ID: **106** (Facebook Ads at 2026-02-20 16:10) ← ONLY ONE ROW
4. Sums `tr_total` for that source

**Result:**

```sql
source_credit  | last_click_revenue
---------------|--------------------
Google Ads     | 0      ($0)
Email          | 0      ($0)
Facebook Ads   | 10000  ($100)  ← Only the last touchpoint
---------------|--------------------
TOTAL          | 10000  ($100)  ← Matches actual order! ✓
```

**This matches Shopify!**

## Why Does Any-Click Exist?

**It's not a bug - it's intentional!** Any-Click answers different questions:

### Any-Click Answers:
> **"Which channels CONTRIBUTED to sales?"**
>
> - Shows every channel that touched a customer before purchase
> - Useful for understanding marketing mix
> - Helps identify assist channels (channels that help but don't close)

**Example:**
- Google Ads introduces customer → Email nurtures → Facebook retargets → Purchase
- Any-Click shows all 3 channels contributed
- You wouldn't want to cut Google Ads even though it didn't get "last click"

### Last-Click Answers:
> **"Which channel CLOSED the sale?"**
>
> - Shows only the final touchpoint before purchase
> - Useful for ROI calculations
> - Matches Shopify revenue (no overcounting)

**Example:**
- Same journey as above
- Last-Click gives 100% credit to Facebook
- You can compare this to ad spend for accurate ROAS

---

## When to Use Each Model

| Model | Best For | Pros | Cons |
|-------|----------|------|------|
| **Any-Click** | Marketing contribution analysis | Shows all channels that helped | Overcounts revenue (can't use for ROAS) |
| **Last-Click** | ROI/ROAS calculations | Matches Shopify revenue | Undervalues assist channels |
| **First-Click** | Customer acquisition analysis | Shows what brought them in | Ignores nurture/retargeting |
| **Linear** | Balanced view | Fair credit distribution | Complex to explain |

## Final Answer to Your Questions

### Q: Does Any-Click double count?

**A: YES!** It gives 100% credit to EVERY touchpoint. An order with 3 touchpoints counts as 3x the revenue.

### Q: How does Last-Click help?

**A: Last-Click only counts each order ONCE** (gives credit to the final touchpoint only). This matches Shopify's total revenue.

### Q: Should we switch to Last-Click?

**A: YES for Revenue Comparison page**, because:
- Users expect it to match Shopify
- Needed for accurate ROAS calculations
- Less confusing

**But keep Any-Click available** for advanced users who want to see channel contribution.

---

**Bottom Line:** Change one line of code (`'Any Click'` → `'Last Click'`) and the $43K discrepancy disappears immediately.
